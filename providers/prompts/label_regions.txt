- Use concise, professional end-user language (2–5 words).
- The label must describe the PURPOSE of the region, not just its type.

- DO NOT write generic labels like "Region 1" or "Unknown".
- Capitalize every word (Title Case).

RULES FOR TYPES:
- filter_form = input fields for searching or filtering data
- action_button = standalone button that triggers an action
- table_header = column headings of a data listing table
- action_column = row-level edit/delete/view action buttons
- navigation_bar = sidebar or top menu navigation links
- page_header = the main title or heading of the page
- section_heading = sub-section title within a page
- view_only = read-only display area

{regions_json}

Return ONLY a JSON array with EXACTLY the same number of items as the input, in the same order.
